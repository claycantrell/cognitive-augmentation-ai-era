# Building Your Manuscript

Using Pandoc with citeproc to turn markdown into publication-ready PDF, DOCX, and HTML.

---

## pandoc + citeproc

Convert markdown to formatted documents with automatic citation processing.

### Install

**macOS:**
```bash
brew install pandoc
```

**Linux:**
```bash
sudo apt install pandoc
```

A LaTeX distribution is required for PDF output. On macOS, install [BasicTeX](https://www.tug.org/mactex/morepackages.html) or full MacTeX. On Linux, `sudo apt install texlive-xetex`.

### Manuscript YAML frontmatter

The file `manuscript/main.md` begins with YAML metadata that controls Pandoc's output:

```yaml
---
title: "Your Title"
author:
  - name: "Author Name"
    affiliation: "Institution"
    email: "email@example.com"
date: "Month Year"
abstract: |
  Your abstract here.
keywords:
  - keyword1
  - keyword2
bibliography: ../bibliography/references.bib
csl: ../bibliography/citation-style.csl
link-citations: true
reference-section-title: References
---
```

Key fields:
- **bibliography** -- path to your `.bib` file (generated by `make bib`).
- **csl** -- citation style file. Download styles from the [Zotero Style Repository](https://www.zotero.org/styles).
- **link-citations** -- makes citations clickable in PDF/HTML output.

### Citation syntax

Use `[@key]` syntax anywhere in your markdown. The `key` matches a BibTeX entry in `references.bib`.

```markdown
# In-text examples

Cognitive load theory [@sweller1988] suggests...

As shown by recent work [@smith2024, p. 42]...

Multiple sources support this [@jones2023; @smith2024].

See @vaswani2017 for the original transformer architecture.

[see @miller2022, pp. 15-20; also @chen2023, ch. 3]
```

| Syntax | Rendered as |
|---|---|
| `[@smith2024]` | (Smith, 2024) |
| `[@smith2024, p. 42]` | (Smith, 2024, p. 42) |
| `[@jones2023; @smith2024]` | (Jones, 2023; Smith, 2024) |
| `@vaswani2017` | Vaswani et al. (2017) |
| `[see @miller2022; also @chen2023]` | (see Miller, 2022; also Chen, 2023) |

### CSL styles

Citation Style Language files control formatting (APA, Chicago, IEEE, etc.). Download from <https://www.zotero.org/styles> and save to `bibliography/citation-style.csl`.

Common styles:

```bash
# APA 7th edition
curl -o bibliography/citation-style.csl \
  "https://raw.githubusercontent.com/citation-style-language/styles/master/apa.csl"

# IEEE
curl -o bibliography/citation-style.csl \
  "https://raw.githubusercontent.com/citation-style-language/styles/master/ieee.csl"

# Chicago Author-Date
curl -o bibliography/citation-style.csl \
  "https://raw.githubusercontent.com/citation-style-language/styles/master/chicago-author-date.csl"
```

### Makefile shortcuts

```bash
# Build PDF (requires LaTeX)
make pdf

# Build DOCX (no LaTeX needed)
make docx

# Build standalone HTML
make html

# Build all three formats
make all

# Word count
make wordcount

# Clean output directory
make clean
```

| Target | Output file | Engine |
|---|---|---|
| `make pdf` | `output/manuscript.pdf` | xelatex via Pandoc |
| `make docx` | `output/manuscript.docx` | Pandoc (native) |
| `make html` | `output/manuscript.html` | Pandoc (standalone) |
| `make all` | all three | all three |

All outputs land in the `output/` directory. The Makefile passes `--citeproc --bibliography=bibliography/references.bib --csl=bibliography/citation-style.csl --pdf-engine=xelatex` to Pandoc automatically.

### How it connects

- Write in `manuscript/main.md` using `[@key]` citations.
- Run `make bib` to regenerate `bibliography/references.bib` from your papis library.
- Run `make pdf` (or `make docx`, `make html`) to compile.
- The bibliography is appended automatically under the "References" heading.
- Use `templates/section-draft.md` to draft individual sections before integrating them into `main.md`.

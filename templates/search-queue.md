# Search Queue

Track what papers you still need to find, organized by section. Claude works through this list during research sessions, updating status as searches are run and papers are found.

## How to use this file

1. **Add searches** as you identify gaps in your argument — what evidence is missing?
2. **Claude works the queue** — running searches, evaluating results, downloading papers, creating reading notes
3. **Update status** after each search session so you never lose track of where you are

## Status Key

- `[ ]` — Not started
- `[-]` — Searched but no good result yet (note what was tried)
- `[~]` — Found candidate(s), not yet downloaded/read
- `[x]` — Done — paper downloaded, note created
